Рекомендуется вести дневник ошибок.
Ошибки заполняются после каждого запуска:
`код -> ошибка -> код -> ок`

#### Где вести дневник ошибок

Советую для этого использовать GitHub/GitLab/Notion/Yonote и хранить все там. можно организовать структуру папок та
```
/algorithms
   RETRY_LIST.md    <- список задачек, которые тяжело шли
   /array           <- имя темы
         ....
   /linked-list     <- имя темы
         /reverse-list <- имя задачи
            fails.md   <- ДНЕВНИК ОШИБОК
         /merge-lists
            fails.md
         ...
      ...
```
#### RETRY_LIST.md

RETRY_LIST.md - это файл, куда стоит записывать все задачки, которые тяжело шли, чтобы потом к ним вернуться. Советую возвращаться к этим задачкам 2 раза:

Первый раз через 2 дня
Второй раз через 4 дня

Тут нет четкого подхода, когда именно возвращаться и ты можешь выбрать для себя другой алгоритм. Главное, вернуться к задачкам, чтобы на собеседовании они не вводили в ступор!
