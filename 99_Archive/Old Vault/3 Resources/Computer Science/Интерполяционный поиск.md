Данный вид поиск пытается предсказать, в какой части массива находится элемент.
В этом он схож с [[Бинарный поиск]], но в отличие от него не делит массив на 2 части, а производит оценку расстояния между ключом и текущим значением.

*Производительность*: $O(log(log \ n))$

Число необходимых операций зависит от равномерности распределения значений среди элементов. В плохом случае (например, когда значения элементов экспоненциально возрастают) интерполяционный поиск может потребовать до $O(N)$ операций.



```swift
public func interpolationSearch(for value: Int, in array: [Int]) -> Int? {
    var low: Int = 0
    var high: Int = array.count - 1
    
    var result: Int? = nil
    while (low <= high && value >= array[low] && value <= array[high]) {
        let pos: Int = low + ((high - low) * (value - array[low])) / (array[high] - array[low])
        
        if array[pos] == value {
            result = pos
            break
        }
        
        if array[pos] < value {
            low = pos + 1
        } else {
            high = pos - 1
        }
        
    }
    return result
}

```
