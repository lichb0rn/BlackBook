Михаил, добрый день.
> 1. rebuild / background verify / migration / snapshot
Ребилд Вы увидете в WEB, если диск отлетит.
Задание проверки также отображается в UI в разделе Activity:
Disk Group: A scrub-disk-group job completed. No errors were found. (disk group: dg01, SN: 00c0ff6672200000a629d96300000000)

Можно также посмотреть из CLI:
show disk-groups scrub-status

scrub - системный процесс, его не рекомендуется выключать.

> 2. перегрузка одного контроллера (ALUA TPG 0 vs 1)

Опять же в GUI это будет, также можно использовать show events


> 1. latency и queue depth по LUN’ам, соответствующим CSV cl3-data2 и cl3-log2;

С метриками сложнее. На массиве не весь доступный набор метрик запущен, чтобы не создавать дополнительную нагрузку.
Список текущих метрик - show metrics-list started
Все доступные - show metrics-list

Вы можете смотреть их командами, которые я уже присылал:

query metrics count all system.read-avg-response-time 
query metrics database historical time-range "start 2026-02-10 10:00 AM end 2026-02-10 11:00 PM" system.read-avg-response-time
